# Правила разметки файлов вопросов (.md)

Для того чтобы ваши файлы вопросов корректно импортировались и разбирались парсером, они должны строго соответствовать структуре, описанной в данном документе.

---

## Общая структура файла

Файл представляет собой набор категорий, внутри которых располагаются вопросы с ответами. Выглядит это следующим образом:

1. **Заголовок первого уровня `#**` (необязательно, игнорируется парсером) — название всего документа.
2. **Разделы/Категории второго уровня `## Название раздела**` — делят вопросы на темы.
3. **Вопросы третьего уровня `### Название вопроса**` — заголовки вопросов, поддерживающие теги и маркеры сложности.

---

## Описание элементов разметки

### 1. Категории (Разделы)

Каждая категория начинается с заголовка второго уровня. Если вопросы идут до первого объявления категории, им автоматически присваивается категория `"Общее"`.

```markdown
## Тема 1. Основы JavaScript

```

Все вопросы, идущие ниже, будут автоматически отнесены к этой категории, пока парсер не встретит следующий заголовок `##`.

### 2. Вопросы, теги и маркеры сложности

Каждый вопрос начинается с заголовка третьего уровня:

```markdown
### [!] 1. Что такое область видимости в JavaScript? #js #basics

```

* **Нумерация**: Если заголовок начинается с числа (например, `1. `), парсер сохранит его как оригинальный номер вопроса. Если номера нет, парсер назначит его автоматически на основе общего количества вопросов.
* **Сложные вопросы `[!]**`: Если добавить метку `[!]` в самое начало заголовка, вопрос будет отмечен флагом повышенной сложности (`isStarred: true`).
* **Теги `#tag**`: Вы можете указывать теги прямо в строке заголовка через символ `#` (поддерживаются латиница, кириллица, цифры, дефисы и подчеркивания). Парсер автоматически удалит их из отображаемого названия вопроса и запишет в массив `tags` в нижнем регистре.

### 3. Ответы (Для вопросов со свободным ответом)

В теле вопроса указываются эталонные ответы в виде списков. Парсер ищет ключевые слова, выделенные жирным шрифтом.

> ⚠️ **Важно**: Каждый новый блок ответа должен начинаться **с новой строки и с маркера списка (`* `)**. Текст ответа извлекается вплоть до следующего маркера ответа или до конца вопроса.

* **Краткий ответ**: Распознается по ключевым словам: `* Краткий ответ:`, `* Код:`, `* Ответ:`.
* **Подробное пояснение**: Распознается по ключевым словам: `* Более расширенный ответ:`, `* Пояснение:`.

#### Пример текстового вопроса:

```markdown
### 3. Какое свойство возвращает длину строки? #strings
* **Краткий ответ**: Свойство `.length`.
* **Пояснение**: Например, для строки `const s = "hi";` выражение `s.length` вернет 2.

```

### 4. Практические вопросы и блоки кода

Парсер автоматически находит **все** блоки кода внутри текста вопроса, оформленные с помощью тройных обратных кавычек. Поддерживаются языки: `javascript`, `typescript`, `html`, `css` (по умолчанию используется `javascript`).

```markdown
### 14. Как удалить последний элемент массива? #arrays
* **Код**:
```javascript
const arr = [1, 2, 3];
arr.pop(); // удаляет и возвращает 3

```

* **Пояснение**: Метод `pop()` изменяет исходный массив.

```

### 5. Вопросы с выбором вариантов (Тесты / Чекбоксы)
Если вопрос содержит варианты ответов, оформите их в виде списка чекбоксов (`- [ ]` или `* [ ]` для неверных и `- [x]` / `* [x]` для верных). Наличие хотя бы одного такого чекбокса автоматически переводит вопрос в тип `choice` (тест).

```markdown
### 5. Какой метод преобразует строку в нижний регистр? #strings
- [ ] toUpper()
- [ ] toLowerCase
- [x] toLowerCase()
- [ ] toLower()

* **Пояснение**: Метод `toLowerCase()` вызывается без аргументов.

```

## Полный пример файла вопросов (Шаблон)

```markdown
# ДЕМО-МОДУЛЬ ДЛЯ ПАРСЕРА

## Часть 1. Переменные

### 1. Чем let отличается от const? #variables #basics
* **Краткий ответ**: Переменную `let` можно переприсвоить, а `const` — нельзя.
* **Пояснение**: Попытка изменить значение `const` приведет к ошибке `TypeError`.

### [!] 2. Что произойдет при обращении к необъявленной переменной? #scoping #errors
* **Краткий ответ**: Возникнет ошибка `ReferenceError`.
* **Пояснение**: Интерпретатор не найдет переменную в текущем лексическом окружении.

## Часть 2. Массивы и Тесты

### 3. Какой метод добавляет элемент в конец массива? #arrays
- [ ] pop()
- [x] push()
- [ ] shift()
- [ ] unshift()

* **Пояснение**: Метод `push()` изменяет исходный массив и возвращает его новую длину.

```
