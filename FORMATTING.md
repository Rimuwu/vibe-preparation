# Правила разметки файлов вопросов (.md)

Для того чтобы ваши файлы вопросов корректно импортировались и разбирались парсером, они должны соответствовать структуре, описанной в данном документе.

---

## Общая структура файла

Файл представляет собой набор категорий, внутри которых располагаются вопросы с ответами. Выглядит это следующим образом:

1. **Заголовок первого уровня `#`** (необязательно, игнорируется парсером) — название всего документа.
2. **Разделы/Категории второго уровня `## Название раздела`** — делят вопросы на темы.
3. **Вопросы третьего уровня `### Название вопроса`** — заголовки вопросов.

---

## Описание элементов разметки

### 1. Категории
Каждая категория начинается с заголовка второго уровня:
```markdown
## Тема 1. Основы JavaScript
```
Все вопросы, идущие ниже, будут автоматически отнесены к этой категории, пока парсер не встретит следующий заголовок `##`.

### 2. Вопросы
Каждый вопрос начинается с заголовка третьего уровня:
```markdown
### 1. Что такое область видимости в JavaScript?
```
* **Нумерация**: Если заголовок начинается с числа (например, `1. `), парсер выделит его как номер вопроса. Если номера нет, парсер назначит его по порядку.
* **Сложные вопросы**: Если добавить метку `[!]` в заголовок вопроса, он будет автоматически отмечен звездочкой как сложный (например, для режима «Только сложные»):
  ```markdown
  ### [!] 12. Что такое замыкание (Closure)?
  ```

### 3. Ответы (Для свободного ответа или ввода текста)
В теле вопроса должны быть указаны эталонные ответы в виде ненумерованного списка.
Парсер ищет ключевые слова, выделенные жирным шрифтом:

* **Краткий ответ**: Распознается по любому из ключевых слов: `* **Краткий ответ**:`, `* **Код**:`, `* **Ответ**:`.
* **Подробное пояснение**: Распознается по ключевым словам: `* **Более расширенный ответ**:`, `* **Пояснение**:`.

#### Пример текстового вопроса:
```markdown
### 3. Какое свойство возвращает длину строки?
* **Краткий ответ**: Свойство `.length`.
* **Пояснение**: Например, для строки `const s = "hi";` выражение `s.length` вернет 2.
```

### 4. Практические вопросы с кодом
Если краткий ответ представляет собой блок кода, оформите его с помощью тройных обратных кавычек с указанием языка:

```markdown
### 14. Как удалить последний элемент массива?
* **Код**:
  ```javascript
  const arr = [1, 2, 3];
  arr.pop(); // удаляет и возвращает 3
  ```
* **Пояснение**: Метод `pop()` изменяет исходный массив.
```

*(Обратите внимание: благодаря исправлению в парсере, пустые строки внутри блока кода больше не приводят к обрезке текста!)*

### 5. Вопросы с выбором вариантов (Тесты)
Если вопрос содержит интерактивный тест с выбором вариантов, оформите варианты в виде списка чекбоксов (`- [ ]` для неверных и `- [x]` для верных):

```markdown
### 5. Какой метод преобразует строку в нижний регистр?
- [ ] toUpper()
- [ ] toLowerCase
- [x] toLowerCase()
- [ ] toLower()

* **Пояснение**: Метод `toLowerCase()` вызывается без аргументов и возвращает новую строку в нижнем регистре.
```

## Полный пример файла вопросов

Вы можете использовать этот пример как шаблон для создания своего модуля:

```markdown
# МОЙ ТЕСТОВЫЙ МОДУЛЬ

## Часть 1. Переменные

### 1. Чем let отличается от const?
* **Краткий ответ**: Переменную `let` можно переприсвоить, а `const` — нельзя.
* **Пояснение**: Попытка изменить значение `const` приведет к ошибке `TypeError`.

### [!] 2. Что произойдет при обращении к необъявленной переменной?
* **Краткий ответ**: Возникнет ошибка `ReferenceError`.
* **Пояснение**: Если переменная не была создана с помощью `let`, `const` или `var`, интерпретатор не найдет ее в текущем лексическом окружении.

## Часть 2. Массивы и Тесты

### 3. Какой метод добавляет элемент в конец массива?
- [ ] pop()
- [x] push()
- [ ] shift()
- [ ] unshift()

* **Пояснение**: Метод `push()` изменяет массив и возвращает его новую длину.
```
