### 1. Почему HTML называют языком разметки, а JavaScript - языком программирования, и в чём принципиальное различие их задач на странице? #html #javascript #basics

* **Краткий ответ**: HTML (HyperText Markup Language) определяет структуру и скелет страницы с помощью тегов, а JavaScript (JS) — это полноценный язык программирования, который управляет логикой, поведением и интерактивностью страницы.
* **Более расширенный ответ**: HTML не содержит алгоритмических конструкций (условий, циклов, функций), он лишь указывает браузеру структуру документа (где заголовок, абзац, кнопка или изображение). JavaScript же является полноценным императивным/мультипарадигмальным языком программирования, который выполняется браузером, содержит управляющую логику, динамически изменяет DOM-дерево, реагирует на действия пользователя и обменивается данными с сервером. Их задачи четко разделены: HTML строит разметку, CSS задает стили, а JavaScript делает страницу «живой» и интерактивной.

- [ ] Способы: локальный и удаленный. JS-файл подключается через <script href="...">, а CSS-файл — через <link src="...">.
- [ ] Способы: через тег <style> и через тег <script>. JS-файл загружается асинхронно по умолчанию, а CSS блокирует выполнение JS.

* **Более расширенный ответ**: Встроенный JS пишется непосредственно в HTML-файле между тегами `<script>...</script>`. Внешний JS выносится в отдельный файл `.js` и загружается через `<script src="path.js"></script>`. Оператор группировки `( )` имеет наивысший приоритет и используется для явного изменения порядка вычислений (например, `(2 + 3) * 4` даст `20` вместо `14`).

- [ ] Оператор % возвращает процентное отношение первого числа ко второму. Применяется для расчета скидок.

* **Краткий ответ**: Оператор `%` возвращает остаток от деления первого числа на второе. Типовая задача — проверка числа на чётность (`num % 2 === 0`).

### 5. Что такое конкатенация и какого типа получится результат при сложении числа со строкой, в отличие от сложения двух чисел? #javascript #types #strings #basics
- [x] Конкатенация — это объединение (склеивание) строк. При сложении числа со строкой получается строка (`string`), а при сложении двух чисел — число (`number`).
- [ ] Конкатенация — это математическое сложение чисел. Результат сложения числа со строкой будет числом, а двух чисел — строкой.
- [ ] Конкатенация — это разделение строки на массив. Результат сложения числа со строкой будет undefined, а двух чисел — null.
- [ ] Конкатенация — это преобразование строки в число. Результат сложения числа со строкой всегда равен NaN.
- [ ] Конкатенация — это сравнение двух строк. Сложение числа со строкой и сложение двух чисел дают тип boolean.

Результатом сложения (конкатенации) станет строка `"105"`. Это происходит из-за перегрузки оператора `+` в JavaScript: если хотя бы один из операндов является строкой, операция интерпретируется как конкатенация строк.

- [ ] Шаблонные строки записываются в одинарных кавычках и ускоряют рендеринг страницы в браузере.
- [ ] Шаблонные строки записываются в угловых скобках < > и позволяют встраивать CSS-стили прямо в строку.
- [ ] Шаблонные строки записываются в фигурных скобках { } и автоматически валидируют email-адреса.

### 7. В чём разница между объявлением переменной и присваиванием ей значения, и почему ключевое слово let указывают только один раз? #javascript #variables #basics
- [x] Объявление переменной резервирует имя переменной в памяти, а присваивание записывает значение в эту область. Ключевое слово `let` указывает интерпретатору создать (объявить) новую переменную в текущей области видимости, поэтому его пишут только один раз при её создании.
- [ ] Объявление создает значение переменной, а присваивание резервирует память. let указывают каждый раз при изменении значения.
- [ ] Объявление удаляет переменную из памяти, а присваивание создает ее. let указывают при каждом обращении к переменной.
- [ ] Объявление и присваивание — это одно и то же действие. let можно указывать сколько угодно раз для одной переменной.
- [ ] Объявление присваивает переменной значение undefined, а присваивание удаляет let. let используется только для констант.
- [ ] Объявление связывает переменную с функцией, а присваивание — с объектом. let можно указывать повторно в глобальной области.

* **Краткий ответ**: Объявление переменной резервирует имя переменной в памяти, а присваивание записывает значение в эту область. Ключевое слово `let` указывает интерпретатору создать (объявить) новую переменную в текущей области видимости, поэтому его пишут только один раз при её создании.
* **Более расширенный ответ**: Объявление (например, `let x;`) сообщает интерпретатору о создании новой переменной в области видимости. Присваивание (например, `x = 10;`) записывает конкретные данные в выделенную область памяти. Дальнейшее изменение значения переменной выполняется без ключевого слова `let`, простым использованием оператора `=`.
### 8. Что такое нотация camelCase и каким требованиям должно отвечать хорошее имя переменной? #javascript #variables #codestyle #basics
- [x] camelCase — это стиль написания составных слов без пробелов, где первое слово пишется с маленькой буквы, а каждое последующее — с большой (например, `myVariableName`). Имя должно быть понятным, осмысленным и отвечать правилам синтаксиса JS.
- [ ] camelCase — это стиль, где слова разделяются дефисами (my-variable-name). Имя должно начинаться с цифры.
- [ ] camelCase — это стиль с нижним подчеркиванием (my_variable_name). Имя должно состоять максимум из 5 символов.
- [ ] camelCase — это стиль, где все буквы заглавные (MYVARIABLENAME). Имя должно содержать спецсимволы.
- [ ] camelCase — это стиль с пробелами между словами. Имя должно соответствовать названию HTML-файла.
- [ ] camelCase — это стиль, где слова пишутся наоборот. Имя переменной должно быть как можно длиннее для читаемости.

* **Краткий ответ**: camelCase — это стиль написания составных слов без пробелов, где первое слово пишется с маленькой буквы, а каждое последующее — с большой (например, `myVariableName`). Имя должно быть понятным, осмысленным и отвечать правилам синтаксиса JS.
* **Более расширенный ответ**: Требования к хорошему имени переменной: 1) Осмысленность и понятность (имя должно отражать суть хранящихся данных, например, `userAge` вместо `a` или `temp`). 2) Соответствие синтаксическим правилам JS (не начинаться с цифры, не содержать спецсимволы кроме `$` и `_`, не совпадать с зарезервированными ключевыми словами вроде `let`, `class`, `function`). 3) Использование общепринятого стиля (camelCase для обычных переменных/функций, PascalCase для классов/компонентов, UPPER_SNAKE_CASE для глобальных констант).
### 9. В чём разница между операторами сравнения == и ===, и почему на практике рекомендуют строгое сравнение? #javascript #operators #types #basics
- [x] Оператор `==` выполняет нестрогое сравнение с неявным приведением типов, а `===` выполняет строгое сравнение без приведения типов (сравниваются и тип, и значение). Строгое сравнение рекомендуют во избежание скрытых багов.
- [ ] == — строгое сравнение по значению и типу, а === — нестрогое с приведением типов. Рекомендуют ==.
- [ ] == сравнивает только типы данных, а === сравнивает только значения. Рекомендуют ===.
- [ ] == используется для сравнения чисел, а === — для сравнения строк. Рекомендуют ==.
- [ ] == возвращает true при равенстве ссылок, а === — при равенстве значений. Рекомендуют ==.
- [ ] == преобразует оба значения в логический тип, а === — в числовой тип. Рекомендуют ==.

* **Краткий ответ**: Оператор `==` выполняет нестрогое сравнение с неявным приведением типов, а `===` выполняет строгое сравнение без приведения типов (сравниваются и тип, и значение). Строгое сравнение рекомендуют во избежание скрытых багов.
* **Более расширенный ответ**: При нестрогом сравнении (`==`) JS пытается привести операнды к общему типу перед сопоставлением (например, `5 == "5"` вернет `true`, `false == 0` также вернет `true`). При строгом сравнении (`===`) типы не приводятся, и если они различаются, сразу возвращается `false` (например, `5 === "5"` вернет `false`). Строгое сравнение делает код предсказуемым и безопасным, исключая неочевидные правила приведения типов в JS.
### 10. Опишите поведение логических операторов И (&&), ИЛИ (||) и НЕ (!) и укажите, при каких значениях операндов && даёт true, а || — false. #javascript #operators #boolean #basics
- [x] Оператор `&&` (И) возвращает `true` только если оба операнда истинны. Оператор `||` (ИЛИ) возвращает `false` только если оба операнда ложны. Оператор `!` (НЕ) инвертирует булево значение.
- [ ] && возвращает true, если хотя бы один операнд истинный. || возвращает false, когда оба истинны. ! выполняет сложение.
