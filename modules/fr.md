## ЧАСТЬ I. Теоретические вопросы

### 1. Почему HTML называют языком разметки, а JavaScript - языком программирования, и в чём принципиальное различие их задач на странице? #html #javascript #basics
* **Краткий ответ**: HTML (HyperText Markup Language) определяет структуру и скелет страницы с помощью тегов, а JavaScript (JS) — это полноценный язык программирования, который управляет логикой, поведением и интерактивностью страницы.
* **Более расширенный ответ**: HTML не содержит алгоритмических конструкций (условий, циклов, функций), он лишь указывает браузеру структуру документа (где заголовок, абзац, кнопка или изображение). JavaScript же является полноценным императивным/мультипарадигмальным языком программирования, который выполняется браузером, содержит управляющую логику, динамически изменяет DOM-дерево, реагирует на действия пользователя и обменивается данными с сервером. Их задачи четко разделены: HTML строит разметку, CSS задает стили, а JavaScript делает страницу «живой» и интерактивной.

### 2. Назовите два способа подключить JavaScript к странице и объясните, чем подключение внешнего JS-файла отличается от подключения CSS. #html #javascript #basics
* **Краткий ответ**: Способы: встроенный (код внутри тега `<script>`) и внешний (через атрибут `src` тега `<script>`). Внешний JS-файл подключается с помощью тега `<script src="...">` и исполняется браузером, а CSS подключается через `<link rel="stylesheet" href="...">` для стилизации.
* **Более расширенный ответ**: Встроенный JS пишется непосредственно в HTML-файле между тегами `<script>...</script>`. Внешний JS выносится в отдельный файл `.js` и загружается через `<script src="path.js"></script>`. Разница с CSS: CSS-файлы подключаются через `<link>` в секции `<head>` и служат для построения CSSOM (дерева стилей), не выполняя логический код. Загрузка JS-файлов по умолчанию блокирует парсинг HTML-документа, так как скрипты могут изменить DOM (для оптимизации используют атрибуты `defer` или `async`), в то время как CSS блокирует только отрисовку страницы до полной загрузки стилей.

### 3. Что такое «операнд», каков приоритет арифметических операторов и зачем нужен оператор группировки? #javascript #operators #basics
* **Краткий ответ**: Операнд — это значение (или переменная), к которому применяется оператор. Приоритет арифметических операторов аналогичен математическому (умножение/деление выше сложения/вычитания). Оператор группировки `()` нужен для изменения этого приоритета.
* **Более расширенный ответ**: В выражении `5 * 2` числа `5` и `2` — это операнды, а `*` — бинарный оператор. Приоритет арифметических операторов в JS идет в следующем порядке: сначала скобки `()`, затем возведение в степень `**`, далее умножение `*`, деление `/` и взятие остатка `%`, а в конце — сложение `+` и вычитание `-`. Оператор группировки `( )` имеет наивысший приоритет и используется для явного изменения порядка вычислений (например, `(2 + 3) * 4` даст `20` вместо `14`).

### 4. Что возвращает оператор взятия остатка % и в какой типовой задаче он применяется? #javascript #operators #basics
* **Краткий ответ**: Оператор `%` возвращает остаток от деления первого числа на второе. Типовая задача — проверка числа на чётность (`num % 2 === 0`).
* **Более расширенный ответ**: Результатом операции `a % b` является остаток, полученный при делении `a` на `b` (например, `8 % 3` вернет `2`, а `5 % 2` вернет `1`). Помимо проверки на чётность/нечётность, этот оператор применяется для циклического перебора элементов в ограниченном диапазоне (например, закольцовывание слайдера по формуле `index % length`), проверки делимости чисел без остатка и ограничения чисел в рамках заданного диапазона (взятие по модулю).

### 5. Что такое конкатенация и какого типа получится результат при сложении числа со строкой, в отличие от сложения двух чисел? #javascript #types #strings #basics
* **Краткий ответ**: Конкатенация — это объединение (склеивание) строк. При сложении числа со строкой получается строка (`string`), а при сложении двух чисел — число (`number`).
* **Более расширенный ответ**: Если сложить два числа `10 + 5`, результатом будет число `15`. Если сложить число со строкой `10 + "5"`, JavaScript выполнит неявное приведение типов (type coercion) и преобразует число `10` в строку `"10"`. Результатом сложения (конкатенации) станет строка `"105"`. Это происходит из-за перегрузки оператора `+` в JavaScript: если хотя бы один из операндов является строкой, операция интерпретируется как конкатенация строк.

### 6. Чем шаблонные строки отличаются от обычных строк в кавычках и какие две дополнительные возможности они дают? #javascript #strings #es6
* **Краткий ответ**: Шаблонные строки записываются в косых кавычках (бекстиках \` \`) и позволяют использовать интерполяцию (вставку выражений) и перенос строк без спецсимволов.
* **Более расширенный ответ**: Обычные строки записываются в одинарных `'` или двойных `"` кавычках. Шаблонные строки дают следующие возможности: 1) Интерполяция переменных и выражений с помощью конструкции `${выражение}` (например, \`Результат: ${a + b}\`). 2) Многострочность — текст можно переносить на новую строку прямо в коде, не используя управляющий символ `\n` и конкатенацию. Также они поддерживают тегирование (tagged templates).

### 7. В чём разница между объявлением переменной и присваиванием ей значения, и почему ключевое слово let указывают только один раз? #javascript #variables #basics
* **Краткий ответ**: Объявление переменной резервирует имя переменной в памяти, а присваивание записывает значение в эту область. Ключевое слово `let` указывает интерпретатору создать (объявить) новую переменную в текущей области видимости, поэтому его пишут только один раз при её создании.
* **Более расширенный ответ**: Объявление (например, `let x;`) сообщает интерпретатору о создании новой переменной в области видимости. Присваивание (например, `x = 10;`) записывает конкретные данные в выделенную область памяти. Если написать `let` для переменной, которая уже была объявлена в этой же области видимости, произойдет ошибка `SyntaxError: Identifier 'x' has already been declared`. Дальнейшее изменение значения переменной выполняется без ключевого слова `let`, простым использованием оператора `=`.

### 8. Что такое нотация camelCase и каким требованиям должно отвечать хорошее имя переменной? #javascript #variables #codestyle #basics
* **Краткий ответ**: camelCase — это стиль написания составных слов без пробелов, где первое слово пишется с маленькой буквы, а каждое последующее — с большой (например, `myVariableName`). Имя должно быть понятным, осмысленным и отвечать правилам синтаксиса JS.
* **Более расширенный ответ**: Требования к хорошему имени переменной: 1) Осмысленность и понятность (имя должно отражать суть хранящихся данных, например, `userAge` вместо `a` или `temp`). 2) Соответствие синтаксическим правилам JS (не начинаться с цифры, не содержать спецсимволы кроме `$` и `_`, не совпадать с зарезервированными ключевыми словами вроде `let`, `class`, `function`). 3) Использование общепринятого стиля (camelCase для обычных переменных/функций, PascalCase для классов/компонентов, UPPER_SNAKE_CASE для глобальных констант).

### 9. В чём разница между операторами сравнения == и ===, и почему на практике рекомендуют строгое сравнение? #javascript #operators #types #basics
* **Краткий ответ**: Оператор `==` выполняет нестрогое сравнение с неявным приведением типов, а `===` выполняет строгое сравнение без приведения типов (сравниваются и тип, и значение). Строгое сравнение рекомендуют во избежание скрытых багов.
* **Более расширенный ответ**: При нестрогом сравнении (`==`) JS пытается привести операнды к общему типу перед сопоставлением (например, `5 == "5"` вернет `true`, `false == 0` также вернет `true`). При строгом сравнении (`===`) типы не приводятся, и если они различаются, сразу возвращается `false` (например, `5 === "5"` вернет `false`). Строгое сравнение делает код предсказуемым и безопасным, исключая неочевидные правила приведения типов в JS.

### 10. Опишите поведение логических операторов И (&&), ИЛИ (||) и НЕ (!) и укажите, при каких значениях операндов && даёт true, а || — false. #javascript #operators #boolean #basics
* **Краткий ответ**: Оператор `&&` (И) возвращает `true` только если оба операнда истинны. Оператор `||` (ИЛИ) возвращает `false` только если оба операнда ложны. Оператор `!` (НЕ) инвертирует булево значение.
* **Более расширенный ответ**: 
  * `&&` (И) вычисляется слева направо и возвращает первое «ложное» (falsy) значение, либо последнее «истинное» (truthy), если все истинны. Дает `true` только когда все операнды приведены к `true`.
  * `||` (ИЛИ) возвращает первое «истинное» значение, либо последнее «ложное», если все ложны. Дает `false` только когда все операнды приведены к `false`.
  * `!` (НЕ) приводит операнд к булевому типу и возвращает противоположное значение (например, `!true` вернет `false`, `!0` вернет `true`).

### 11. Какие значения в JavaScript считаются «ложными» (falsy) и как конструкция if интерпретирует не-булево значение, переданное в условие? #javascript #types #boolean #basics
* **Краткий ответ**: «Ложные» (falsy) значения (8): `false`, `0`, `-0`, `0n` (BigInt), `""` (пустая строка), `null`, `undefined` и `NaN`. Любые другие значения приводятся к `true`. Конструкция `if` неявно преобразует переданное значение в булево перед проверкой.
* **Более расширенный ответ**: Когда в `if (условие)` передаётся не-булевое значение, JS автоматически выполняет его приведение к логическому типу (эквивалентно вызову функции `Boolean(условие)`). Если значение является одним из 8 falsy-значений, условие считается ложным, и блок кода внутри `if` не выполняется. Все остальные значения (включая пустые объекты `{}` и пустые массивы `[]`) приводятся к `true`, и управление передается внутрь блока `if`.

### 12. Что такое массив, с какого индекса начинается нумерация его элементов и какое свойство возвращает их количество? #javascript #arrays #basics
* **Краткий ответ**: Массив — это упорядоченная структура данных для хранения списка элементов. Нумерация начинается с `0`, а свойство `length` возвращает их количество.
* **Более расширенный ответ**: В JavaScript массивы являются особым типом объектов (тип `object` при проверке `typeof`), оптимизированным для работы с упорядоченными коллекциями данных. Доступ к элементам осуществляется по числовым индексам через квадратные скобки (например, `arr[0]`). Свойство `length` возвращает текущее количество элементов в массиве (индекс последнего элемента плюс один) и может быть изменено вручную для усечения массива.

### 13. Сравните циклы for, while и do...while и объясните, чем do...while принципиально отличается от while. #javascript #loops #basics
* **Краткий ответ**: `for` используется при известном количестве итераций. `while` проверяет условие до выполнения итерации. `do...while` проверяет условие после итерации, гарантируя хотя бы один запуск тела цикла.
* **Более расширенный ответ**:
  * `for` объединяет в своей сигнатуре инициализацию, условие и шаг цикла (удобен для перебора массивов).
  * `while (условие)` проверяет истинность условия перед началом каждого шага. Если условие ложно изначально, цикл не выполнится ни разу.
  * `do { тело } while (условие)` является циклом с постусловием. Тело цикла выполняется сначала, а проверка условия происходит только в конце шага. Таким образом, цикл `do...while` гарантированно выполнится как минимум один раз, независимо от того, истинно условие или нет на старте.

### 14. Что такое функция и в чём разница между её параметрами и аргументами, передаваемыми при вызове? #javascript #functions #basics
* **Краткий ответ**: Функция — это именованный блок кода, который можно вызывать многократно. Параметры — это переменные-плейсхолдеры, указанные в объявлении функции. Аргументы — это реальные значения, которые передаются в функцию при её вызове.
* **Более расширенный ответ**: Функция позволяет инкапсулировать логику для повторного использования. При создании функции мы объявляем её параметры: `function greet(name, age) { ... }`, где `name` и `age` — это локальные переменные (параметры). При вызове функции мы передаем конкретные значения (аргументы): `greet("Алексей", 20)`. Аргументы копируются в соответствующие параметры на время выполнения функции.

### 15. Каково назначение оператора return, что вернёт функция без него и как return влияет на дальнейшее выполнение функции? #javascript #functions #basics
* **Краткий ответ**: Оператор `return` завершает выполнение функции и возвращает результат её работы. Функция без `return` возвращает `undefined`. Любой код, расположенный после `return` в теле функции, игнорируется.
* **Более расширенный ответ**: Оператор `return` используется для передачи вычисленного значения обратно в точку вызова функции. Как только интерпретатор доходит до ключевого слова `return`, выполнение функции немедленно прекращается. Если `return` вызван без выражения или отсутствует в функции вовсе, функция завершится после выполнения последней строки кода и вернет значение `undefined`.

### 16. Что такое объект в JavaScript, чем он отличается от массива и в каких случаях его удобнее использовать? #javascript #objects #basics
* **Краткий ответ**: Объект — это неупорядоченная структура данных, хранящая пары "ключ: значение". Массив хранит упорядоченные данные по числовым индексам. Объект удобнее использовать для описания сложных сущностей со свойствами (например, пользователь с именем, почтой и возрастом).
* **Более расширенный ответ**: Объект объявляется с помощью фигурных скобок `{}` и содержит именованные свойства (ключи являются строками или символами). Массив предназначен для упорядоченных списков однотипных элементов, где важен порядок следования и доступ осуществляется по числам. Объекты удобнее использовать, когда нам нужно хранить модель реального объекта с именованными характеристиками, а не просто перечень элементов (например, `const car = { brand: "Toyota", speed: 180 }`).

### [!] 17. Перечислите примитивные типы данных и объясните, для чего предназначен оператор typeof и что он возвращает. #javascript #types #primitives
* **Краткий ответ**: Примитивные типы (7): `number`, `string`, `boolean`, `null`, `undefined`, `symbol`, `bigint`. Оператор `typeof` определяет тип значения и возвращает его название в виде строки.
* **Более расширенный ответ**: Примитивные значения хранятся непосредственно в ячейках памяти и являются неизменяемыми (immutable). Оператор `typeof` используется для динамического определения типа данных операнда во время исполнения программы. Возвращаемые значения: `"number"`, `"string"`, `"boolean"`, `"undefined"`, `"symbol"`, `"bigint"`, `"function"` (для функций) и `"object"` (для объектов, массивов, а также исторически для `null`).

### 18. В чём смысловое различие между undefined и null и в каких ситуациях возникает каждое из этих значений? #javascript #types #null #undefined
* **Краткий ответ**: `undefined` означает, что значение переменной не было присвоено (отсутствие по умолчанию). `null` означает намеренное отсутствие значения (пустое значение, заданное разработчиком).
* **Более расширенный ответ**: 
  * `undefined` возникает автоматически: при объявлении переменной без инициализации (`let a;`), при обращении к несуществующему свойству объекта, при отсутствии возвращаемого значения в функции или при непередаче аргумента в параметр.
  * `null` — это специальное литеральное значение, которое присваивается переменной или свойству вручную, чтобы явно указать, что значение пусто или временно отсутствует (например, `let currentUser = null;` до момента авторизации).

### [!] 19. Что такое тип Symbol и какое его главное свойство делает его полезным в роли уникального идентификатора? #javascript #types #symbol #advanced
* **Краткий ответ**: `Symbol` (символ) — это уникальный примитивный тип данных. Его главное свойство — гарантированная уникальность: каждый созданный символ уникален, даже если у них одинаковые описания.
* **Более расширенный ответ**: Символ создается с помощью вызова функции `Symbol("описание")`. Выражение `Symbol("id") === Symbol("id")` вернет `false`. Это свойство делает символы крайне полезными для создания скрытых свойств объектов, которые не могут быть случайно перезаписаны другими частями программы или сторонними библиотеками, а также для создания системных хуков внутри объектов (например, `Symbol.iterator`).

### 20. Как узнать длину строки и как обратиться к отдельному символу строки по индексу? Назовите свойство и способ доступа. #javascript #strings #basics
* **Краткий ответ**: Длина строки возвращается свойством `.length`. Обратиться к отдельному символу можно по индексу через квадратные скобки `[индекс]` или методом `.charAt(индекс)`.
* **Более расширенный ответ**: Для строки `const str = "Frontend"` выражение `str.length` вернет число `8`. Нумерация символов начинается с `0`, поэтому первый символ доступен как `str[0]`, а последний — как `str[str.length - 1]`. Разница между `str[index]` и `str.charAt(index)` в том, что при выходе за границы строки квадратные скобки вернут `undefined`, а метод `charAt` — пустую строку `""`.

### 21. Что такое специальные числовые значения NaN и Infinity, для чего служат методы Number.isNaN и Number.isFinite, и чем тип BigInt отличается от обычного number? #javascript #types #number
* **Краткий ответ**: `NaN` (Not-a-Number) возникает при невалидных математических операциях. `Infinity` представляет бесконечность. `Number.isNaN` проверяет, является ли значение точно `NaN`, а `Number.isFinite` проверяет, является ли число конечным. `BigInt` предназначен для целых чисел произвольной длины, превышающих предел безопасных чисел типа `number`.
* **Более расширенный ответ**:
  * `NaN` возвращается при попытке выполнить бессмысленную операцию, например `"text" / 2`. `Infinity` получается при делении ненулевого числа на ноль или при переполнении лимитов.
  * Сравнение `x === NaN` всегда ложно (даже `NaN === NaN` дает `false`). Поэтому метод `Number.isNaN(x)` проверяет, является ли значение именно `NaN`. `Number.isFinite(x)` возвращает `true`, если значение не равно `NaN`, `Infinity` или `-Infinity`.
  * Обычный тип `number` ограничен диапазоном безопасных целых чисел от $-(2^{53} - 1)$ до $2^{53} - 1$. `BigInt` (создаваемый с суффиксом `n`, например `9007199254740991n`) позволяет производить точные операции с целыми числами любого размера, но не может смешиваться с обычным `number` без явного приведения.

### [!] 22. Что такое DOM и как HTML-страница представлена в виде DOM-дерева? #javascript #dom #basics
* **Краткий ответ**: DOM (Document Object Model) — это программный интерфейс, представляющий структуру HTML-документа в виде иерархического дерева объектов, доступных для изменения из JavaScript.
* **Более расширенный ответ**: Браузер парсит HTML-код страницы и строит в памяти дерево объектов. Корнем дерева является глобальный объект `document`. Каждый HTML-тег (например, `<body>`, `<div>`) становится узлом-элементом (Element Node). Содержимое тегов становится текстовыми узлами (Text Node), а HTML-комментарии — узлами комментариев. JavaScript может динамически обращаться к узлам этого дерева, изменять их содержимое, структуру, стили, а также удалять и добавлять новые элементы.

### 23. Какими способами можно искать элементы в DOM и чем querySelector отличается от querySelectorAll, что возвращает каждый из них? #javascript #dom #basics
* **Краткий ответ**: Способы поиска: по CSS-селекторам (`querySelector`, `querySelectorAll`), по ID (`getElementById`), по классу (`getElementsByClassName`), по тегу (`getElementsByTagName`). Метод `querySelector` возвращает первый найденный элемент, а `querySelectorAll` — статическую коллекцию всех найденных элементов.
* **Более расширенный ответ**: Современный подход рекомендует использовать методы, принимающие CSS-селекторы.
  * `document.querySelector(selector)` ищет элемент сверху вниз по дереву DOM и возвращает первый совпавший объект Element или `null`, если совпадений нет.
  * `document.querySelectorAll(selector)` находит все совпадающие элементы и возвращает неживую (статическую) коллекцию `NodeList`, которую можно обходить, например, методом `.forEach()`.

### 24. Опишите назначение методов getAttribute, setAttribute, hasAttribute и removeAttribute для работы с атрибутами элементов. #javascript #dom #attributes
* **Краткий ответ**: `getAttribute(name)` возвращает значение атрибута, `setAttribute(name, value)` создает или перезаписывает атрибут, `hasAttribute(name)` проверяет наличие атрибута, `removeAttribute(name)` удаляет его.
* **Более расширенный ответ**: Эти методы работают непосредственно с HTML-атрибутами (строками в разметке тега). Например, `elem.setAttribute('type', 'password')` изменит тип инпута в HTML. `elem.getAttribute('href')` вернет точное строковое значение ссылки. Методы полезны для работы с кастомными дата-атрибутами (`data-*`), так как они позволяют гибко считывать и записывать текстовые параметры непосредственно в разметку элементов.

### [!] 25. Что такое classList и что делают его методы add, remove, contains и toggle? #javascript #dom #classlist #css
* **Краткий ответ**: `classList` — это свойство элемента, представляющее собой специальный объект-коллекцию для удобного управления его CSS-классами.

  * `remove(className)`: удаляет класс.
  * `contains(className)`: проверяет, есть ли класс у элемента (возвращает `true/false`).
  * `toggle(className)`: переключает класс (удаляет, если он есть, и добавляет, если его нет).

  * `add` и `remove` безопасны: если добавляемый класс уже есть, он не продублируется.
  * `toggle` может принимать второй необязательный логический аргумент (force), который принудительно включает или выключает класс в зависимости от значения выражения.

### 26. В чём разница между свойствами innerHTML и textContent и какой риск безопасности возникает при вставке через innerHTML данных от пользователя? #javascript #dom #security #xss
* **Краткий ответ**: `innerHTML` считывает или записывает содержимое элемента в виде HTML-кода (интерпретируя теги), а `textContent` работает только с чистым текстом (экранируя теги). Использование `innerHTML` для вставки данных от пользователей создает риск XSS-атак (Cross-Site Scripting).
* **Более расширенный ответ**: Если записать `elem.innerHTML = "<b>Привет</b>"`, на странице отобразится жирный текст. Если записать `elem.textContent = "<b>Привет</b>"`, теги отобразятся на экране как обычный текст. Если злоумышленник введет строку с вредоносным кодом (например, `<img src="x" onerror="sendDataToHacker(document.cookie)">`) и мы вставим её через `innerHTML`, браузер выполнит данный JavaScript-код. Вставка через `textContent` абсолютно безопасна, так как все теги экранируются.

### 27. Какими методами создаются, добавляются и удаляются элементы DOM и для чего нужен элемент template? #javascript #dom #elements
* **Краткий ответ**: Создание: `document.createElement(tagName)`. Добавление: `.append()`, `.prepend()`, `.before()`, `.after()`. Удаление: `.remove()`. Тег `<template>` служит невидимым контейнером для хранения HTML-шаблонов, которые можно динамически клонировать и наполнять данными в JS.
* **Более расширенный ответ**:
  * `document.createElement('div')` генерирует новый узел-элемент в памяти.
  * Метод `parent.append(child)` добавляет элемент в конец списка детей родителя. `prepend` вставляет в начало. `before` и `after` вставляют элемент до или после целевого узла.
  * `elem.remove()` полностью вырезает элемент из DOM-дерева.
  * Тег `<template>` парсится браузером, но его содержимое не отображается и не загружает ресурсы (картинки, скрипты), пока мы не клонируем его контент в JavaScript с помощью `template.content.cloneNode(true)` для последующей вставки в документ.

### 28. Что такое событие в браузере, каково назначение метода addEventListener и что содержит объект event (в частности, свойство target)? #javascript #events #dom
* **Краткий ответ**: Событие — это сигнал от браузера о том, что на странице произошло действие (клик, ввод, прокрутка). `addEventListener` регистрирует функцию-обработчик этого события. Объект `event` хранит информацию о событии, а свойство `event.target` указывает на элемент, который непосредственно вызвал это событие.
* **Более расширенный ответ**: Метод `addEventListener(type, listener)` позволяет вешать произвольное количество слушателей на одно событие без перезаписи предыдущих. Объект события `event` (передается первым аргументом в колбэк) содержит метаданные о произошедшем: тип события (`type`), координаты клика, нажатые клавиши. Свойство `event.target` ссылается на целевой элемент, на котором событие произошло (например, конкретный тег `<span>` внутри кнопки), в то время как `event.currentTarget` указывает на элемент, на котором сработал сам обработчик (например, сама кнопка).

### [!] 29. Что делает метод preventDefault() и у какого события есть отменяемое поведение по умолчанию? #javascript #events #dom #forms
* **Краткий ответ**: Метод `preventDefault()` отменяет стандартные действия, которые браузер выполняет по умолчанию при наступлении определенного события. Отменяемое поведение есть, например, у событий `submit` (отправка формы с перезагрузкой страницы) и `click` по ссылке (переход на другой URL).
* **Более расширенный ответ**: Вызов `event.preventDefault()` сообщает браузеру, что стандартное действие не должно выполняться. Это используется, когда мы хотим обработать действие пользователя полностью через JavaScript (например, отправить форму фоновым AJAX-запросом или предотвратить открытие контекстного меню браузера при событии `contextmenu`). Чтобы поведение было отменяемым, свойство события `event.cancelable` должно быть равно `true`.

### [!] 30. Что такое делегирование событий и какие преимущества оно даёт при большом количестве элементов или при их динамическом добавлении? #javascript #events #patterns #advanced
* **Краткий ответ**: Делегирование событий — это паттерн, при котором один обработчик вешается на общего родителя, а целевые элементы определяются внутри через `event.target`. Это существенно экономит оперативную память и автоматически обрабатывает клики на динамически добавляемых элементах.
* **Более расширенный ответ**: Паттерн основан на механизме всплытия событий (bubbling) в DOM. Когда событие происходит на дочернем элементе, оно поднимается вверх к родителю. Вместо того чтобы вешать обработчики на 100 кнопок списка, мы вешаем один обработчик на контейнер `<ul>`. Внутри обработчика мы проверяем: `const button = event.target.closest('.btn')`. Если клик был по кнопке, выполняем логику. Преимущества: 1) Требуется меньше памяти (всего один слушатель). 2) Не нужно вешать обработчики на новые элементы, добавляемые в DOM динамически.

### [!] 31. Как устроен событийный цикл: что такое стек вызовов, очередь макрозадач и Web APIs? #javascript #async #event-loop #advanced
* **Краткий ответ**: Событийный цикл (Event Loop) координирует выполнение кода. Стек вызовов (Call Stack) выполняет синхронный код. Web APIs предоставляют браузерные возможности (таймеры, запросы). Очередь макрозадач (Callback Queue) хранит колбэки асинхронных операций, ожидающие выполнения после очистки стека.
* **Более расширенный ответ**:
  * **Стек вызовов (Call Stack)** — структура данных, работающая по принципу LIFO (последним пришёл — первым обслужен). В ней хранятся выполняемые синхронные функции.
  * **Web APIs** — фоновые потоки браузера, выполняющие операции вне основного потока JS (таймеры `setTimeout`, сетевые запросы `fetch`, события мыши). После завершения они отправляют свои колбэки в очередь.
  * **Очередь задач (Callback Queue)** — очередь FIFO (первым пришёл — первым обслужен) для колбэков.
  * **Event Loop** непрерывно проверяет: если Call Stack пуст, он берет первую задачу из очереди и переносит ее в стек вызовов для выполнения.

### 32. Как из JavaScript получить доступ к форме и её полям и в чём разница между событиями change и input, а также между методами формы reset и submit? #javascript #forms #dom #events
* **Краткий ответ**: Доступ к форме осуществляется через коллекцию `document.forms` или поиском по селектору, а к полям — по их имени (`form.elements.name`). Событие `input` срабатывает мгновенно при каждом изменении содержимого поля, а `change` — при потере фокуса. Метод `submit()` отправляет форму на сервер, а `reset()` сбрасывает её поля.
* **Более расширенный ответ**:
  * Поля формы доступны по атрибутам `name`: `form.elements.username` или напрямую `form.username`.
  * Событие `input` генерируется сразу при вводе символа, удалении текста или вставке. Событие `change` для текстовых полей срабатывает, только когда пользователь изменил значение и убрал фокус с элемента (событие `blur`). Для чекбоксов и селектов `change` срабатывает сразу.
  * Вызов `form.reset()` возвращает значения полей к дефолтным (указанным в HTML). Вызов `form.submit()` отправляет форму программно (при этом браузерное событие `submit` на форме не генерируется, и встроенная валидация HTML5 не запускается).

### 33. В чём разница между localStorage и sessionStorage и какого типа данные в них хранятся? #javascript #storage #browser
* **Краткий ответ**: Оба хранят данные в виде строк по парам "ключ: значение". Разница в жизненном цикле: данные в `localStorage` сохраняются после закрытия браузера и хранятся бессрочно, а в `sessionStorage` удаляются сразу после закрытия вкладки.

  * `sessionStorage` изолирован в рамках конкретной вкладки браузера: открытие той же страницы во второй вкладке создаст новую независимую сессию. `localStorage` доступен во всех вкладках одного домена одновременно.

### 35. Чем различаются ошибки SyntaxError, ReferenceError и TypeError? Приведите пример ситуации, в которой возникает каждая из них. #javascript #errors #debugging
* **Краткий ответ**:
  * `SyntaxError`: ошибка синтаксиса (код нарушает правила грамматики JS и не может быть распарсен).
  * `ReferenceError`: обращение к переменной, которая не была объявлена.
  * `TypeError`: попытка выполнить операцию с несовместимым типом данных.
* **Более расширенный ответ**:
  * **SyntaxError**: код не парсится компилятором. Пример: `if (x {` (пропущена закрывающая скобка условия).
  * **ReferenceError**: обращение к имени, которого нет в текущих лексических областях видимости. Пример: `console.log(nonExistentVar);` или попытка обратиться к переменной, объявленной через `let/const`, до её объявления (в "мертвой зоне").
  * **TypeError**: значение существует, но операция для него недопустима. Пример: попытка вызвать число как функцию (`const x = 5; x();`) или обратиться к свойству значения `null`/`undefined` (`null.toString()`).

### 36. Чем логические ошибки опаснее синтаксических, почему их сложнее обнаружить и как в их поиске помогает console.log? #javascript #errors #debugging
* **Краткий ответ**: Синтаксические ошибки ломают выполнение программы сразу и выводят понятное сообщение в консоль. Логические ошибки не вызывают сбоев программы, но приводят к неверному поведению алгоритма. `console.log` помогает отслеживать промежуточные значения переменных в ключевых точках для локализации сбоя.
* **Более расширенный ответ**: Логическая ошибка — это ошибка в самом алгоритме (например, вместо `<` написали `>`, или перепутали знаки сложения и вычитания). Так как код написан синтаксически корректно, интерпретатор выполняет его без ошибок, но результаты работы программы оказываются неверными. Обнаружить их сложно, так как нет логов об ошибках. Вывод значений через `console.log(variable)` на промежуточных этапах выполнения позволяет увидеть, на каком именно шаге реальный ход вычислений начинает расходиться с ожидаемым.

### 37. Что такое инструкция debugger и чем отладка через точки останова бывает удобнее вывода значений через console.log? #javascript #debugging #devtools
* **Краткий ответ**: Инструкция `debugger` приостанавливает выполнение JavaScript-кода в точке вызова (как breakpoint), если открыты инструменты разработчика DevTools. Отладка удобнее `console.log`, так как позволяет исследовать текущее состояние всех переменных и пошагово проходить по коду в реальном времени.
* **Более расширенный ответ**: При использовании `console.log` разработчик вынужден постоянно модифицировать исходный код и перезапускать приложение. Инструкция `debugger` (или обычные точки останова в панели Sources) позволяет: 1) Приостановить выполнение программы на определенной строке. 2) Посмотреть текущую область видимости (Scope) и стек вызовов (Call Stack). 3) Выполнять код по шагам (Step Over, Step Into, Step Out) и интерактивно вычислять выражения в консоли в текущем контексте выполнения.

### 38. Что такое IIFE и какую проблему она помогает решить по сравнению с использованием глобальных переменных? #javascript #patterns #scope #advanced
* **Краткий ответ**: IIFE (Immediately Invoked Function Expression) — это функция, которая вызывается мгновенно сразу после её объявления. Она решает проблему изоляции области видимости и предотвращает загрязнение глобального пространства имен.
* **Более расширенный ответ**: Синтаксис IIFE: `(function() { let privateVar = 5; })();`. Переменные, объявленные внутри IIFE, остаются локальными для этой функции и недоступны извне. До появления ES-модулей (`import/export`) и блочной области видимости (`let`, `const`), IIFE была основным паттерном инкапсуляции кода в JavaScript, предотвращавшим конфликты имен, когда разные скрипты и библиотеки использовали одинаковые глобальные переменные.

### 39. Что такое инкапсуляция и как модули в JavaScript помогают её обеспечить? #javascript #modules #oop #advanced
* **Краткий ответ**: Инкапсуляция — это сокрытие внутренней реализации и данных объекта/модуля, предоставляя наружу только контролируемый публичный интерфейс. Модули JS изолируют область видимости файлов по умолчанию, экспортируя только то, что явно указано.
* **Более расширенный ответ**: В ES-модулях (`type="module"`) каждый файл имеет свою собственную изолированную область видимости. Другие части программы могут использовать только те элементы, которые были импортированы, не имея возможности напрямую вмешаться во внутреннее состояние и приватную логику модуля.

### 40. Объясните разницу между именованным экспортом и экспортом по умолчанию и чем отличается синтаксис их импорта. #javascript #modules #es6

  * **Именованный экспорт**: `export const foo = 1; export const bar = 2;`. При импорте мы обязаны использовать фигурные скобки и точные имена: `import { foo, bar } from './module.js'`.

* **Краткий ответ**: Директивы `as` используются для переименования сущностей при экспорте или импорте. Это необходимо для предотвращения конфликтов имён в текущем файле или улучшения читаемости API.

### 43. Что такое сборка кода, чем отличаются сборщики Webpack и Vite и что такое tree shaking? #tooling #vite #webpack #build
* **Краткий ответ**: Сборка кода — это объединение, трансляция и оптимизация файлов проекта в итоговые файлы (бандлы) для браузера. Webpack собирает проект, строя полный граф зависимостей перед запуском, а Vite использует нативный ESM в браузере для моментального dev-старта. * **Более расширенный ответ**:
  * Webpack анализирует все файлы, транспилирует их и собирает монолитный бандл. На больших проектах старт dev-сервера может занимать много времени.
  * Vite во время разработки отдает файлы напрямую браузеру через нативные ES-модули, а компиляцию зависимостей производит очень быстро с помощью esbuild. Для продакшн-сборки Vite использует оптимизированный сборщик Rollup.
  * **Tree shaking** (встряхивание дерева) — оптимизация, которая анализирует конструкции `import` и `export`. Если какая-то функция была экспортирована, но не используется в импортах других файлов, сборщик исключает её из итогового файла бандла, уменьшая его вес.

### 46. Для чего используется атрибут novalidate и зачем отключать встроенную валидацию, если планируется реализовать её на JavaScript? #html #forms #validation

### [!] 47. Что такое регулярное выражение и каково назначение якорей ^ и $, наборов символов ([a-z]) и квантификаторов *, +, ? #javascript #regex #advanced
* **Краткий ответ**: Регулярное выражение (RegExp) — это шаблон поиска и проверки совпадения символов в тексте.
  * `^` и `$`: якоря начала и конца строки соответственно.
  * `[a-z]`: класс (набор) символов, соответствующий любой строчной латинской букве.
  * `*`: квантификатор "0 или более повторений".
  * `+`: квантификатор "1 или более повторений".
  * `?`: квантификатор "0 или 1 повторение" (делает символ необязательным).
* **Более расширенный ответ**: Регулярные выражения в JS создаются в косых чертах `/шаблон/` или через `new RegExp()`. Якоря `^` и `$` гарантируют соответствие шаблона всей строке, а не её части. Набор `[a-zA-Z]` позволяет задавать диапазоны разрешенных символов. Квантификаторы задают количество повторений предыдущего символа или группы. Например, `/^\d{2,4}$/` будет соответствовать строке, состоящей строго из 2, 3 или 4 цифр.

  * Флаги: `g` (глобальный поиск), `i` (регистронезависимость), `m` (многострочный поиск), `u` (поддержка Юникода).

  * Альтернация `|` означает логическое ИЛИ.
  * Экранирование `\` превращает спецсимволы в обычные текстовые символы.

### 49. В чём разница между синхронным и асинхронным кодом и зачем во фронтенде нужен асинхронный код? #javascript #async #basics
* **Краткий ответ**: Синхронный код выполняется последовательно, строка за строкой, блокируя выполнение последующих строк до завершения текущей. Асинхронный код позволяет выполнять длительные операции в фоновом режиме, не блокируя основной поток выполнения. Он необходим во фронтенде, чтобы интерфейс сайта оставался отзывчивым во время загрузки данных или работы таймеров.
* **Более расширенный ответ**: JavaScript однопоточен. Если выполнять тяжелые задачи (сетевые запросы, чтение файлов) синхронно, вкладка браузера зависнет: пользователь не сможет нажимать на кнопки, вводить текст или прокручивать страницу до тех пор, пока запрос не завершится. Асинхронный код делегирует выполнение этих операций среде окружения (браузеру/Web API), а после их завершения обрабатывает результат в основном потоке через колбэки, сохраняя отзывчивость интерфейса.

### 50. Что такое колбэк-функция, чем синхронный колбэк отличается от асинхронного и каким примером асинхронного колбэка может служить загрузка изображения? #javascript #async #callbacks
* **Краткий ответ**: Колбэк — это функция, переданная в другую функцию в качестве аргумента для её последующего вызова. Синхронный колбэк выполняется немедленно (например, в методе `.map()`). Асинхронный колбэк выполняется позже, при наступлении асинхронного события (например, событие `onload` при загрузке картинки).
* **Более расширенный ответ**:
  * Синхронный: `[1,2].forEach(x => console.log(x))` — анонимная функция вызывается немедленно в процессе итерации.
  * Асинхронный: Мы создаем объект изображения в JS: `const img = new Image()`. Мы вешаем обработчик события загрузки: `img.onload = () => console.log('Изображение загружено')`, после чего присваиваем адрес `img.src = 'photo.jpg'`. Код, расположенный ниже, продолжит выполняться сразу же, а функция-колбэк сработает только тогда, когда картинка полностью скачается из сети.

### 51. Сравните setTimeout и setInterval: что они возвращают и как остановить запланированное выполнение? #javascript #async #timers
* **Краткий ответ**: `setTimeout` выполняет функцию один раз после заданной задержки. `setInterval` выполняет функцию регулярно через равные промежутки времени. Оба метода возвращают уникальный числовой идентификатор таймера. Остановить выполнение можно методами `clearTimeout(id)` и `clearInterval(id)`.
* **Более расширенный ответ**:
  * Синтаксис: `const timerId = setTimeout(callback, 2000);`.
  * Возвращаемый `timerId` — это число, идентифицирующее созданный таймер в системе браузера. Если мы вызовем `clearTimeout(timerId)` до истечения 2 секунд, то переданная функция `callback` никогда не выполнится. Аналогично, вызов `clearInterval(intervalId)` останавливает бесконечный цикл запусков функции в `setInterval`.

### 52. Как Event Loop выполняет асинхронный код (роль колстека и очереди колбэков) и почему setTimeout с задержкой 0 сработает не мгновенно? #javascript #async #event-loop #advanced
* **Краткий ответ**: Асинхронные колбэки помещаются в очередь задач. Event Loop переносит их в стек вызовов (Call Stack) только тогда, когда он полностью освободится от синхронного кода. `setTimeout(..., 0)` планирует выполнение на ближайший момент, но выполнится только после завершения всего текущего синхронного кода.
* **Более расширенный ответ**: При вызове `setTimeout(callback, 0)` браузер регистрирует таймер, который завершается мгновенно, и колбэк сразу отправляется в очередь макрозадач. Однако Event Loop работает по строгому правилу: он не переносит задачи из очереди в стек вызовов, пока в стеке выполняется хотя бы одна синхронная строчка кода. Таким образом, даже с задержкой 0 мс колбэк таймера выполнится строго после того, как завершится весь синхронный код текущего файла.

### 53. Что такое Promise? Перечислите три состояния промиса и опишите переходы между ними. #javascript #async #promises
* **Краткий ответ**: `Promise` (промис) — это объект, представляющий результат асинхронной операции, которая завершится в будущем.
  Три состояния:
  1. `pending` (ожидание) — начальное состояние.
  2. `fulfilled` (выполнено успешно).
  3. `rejected` (выполнено с ошибкой).
  Переходы: из `pending` в `fulfilled` (через вызов `resolve()`) или из `pending` в `rejected` (через вызов `reject()`). Переход окончателен.
* **Более расширенный ответ**: Создается промис через конструктор: `new Promise((resolve, reject) => { ... })`. Переходы состояний являются однократными и необратимыми. Если промис перешел в состояние успешного завершения (`fulfilled`) или ошибки (`rejected`), он фиксирует свой результат навсегда. Любые последующие попытки вызвать `resolve` или `reject` будут проигнорированы, что гарантирует стабильность и предсказуемость получения асинхронного результата.

### 54. Как строится цепочка обработки промиса через .then() и .catch(), что возвращает .then() и для чего нужен метод Promise.all (что будет при ошибке одного из промисов)? #javascript #async #promises #advanced
* **Краткий ответ**: Цепочка строится последовательными вызовами `.then(onSuccess)` для успешного результата и завершается `.catch(onError)` для обработки ошибок. Метод `.then()` всегда возвращает новый промис. `Promise.all` запускает массив промисов параллельно; если хотя бы один из них упадет с ошибкой, весь `Promise.all` мгновенно отклоняется с этой ошибкой.
* **Более расширенный ответ**:
  * Метод `.then()` возвращает новый промис, значением которого становится результат выполнения колбэка. Если колбэк возвращает другой промис, то внешний промис будет ожидать его завершения. Это позволяет строить плоские цепочки асинхронных шагов вместо вложенных колбэков (callback hell).
  * `Promise.all([p1, p2])` возвращает один промис, который успешно разрешается массивом результатов только тогда, когда завершатся все переданные промисы. При возникновении ошибки на любом этапе, выполнение остальных промисов не прерывается автоматически, но итоговый промис `Promise.all` немедленно переходит в состояние `rejected`, игнорируя результаты остальных успешных промисов.

### 55. Что такое протокол HTTP и из каких частей состоят HTTP-запрос и HTTP-ответ (включая заголовки и тело)? #http #network #basics
* **Краткий ответ**: HTTP (HyperText Transfer Protocol) — это протокол прикладного уровня передачи данных для обмена информацией между клиентом и сервером.
  * HTTP-запрос состоит из: стартовой строки (метод, URL, версия протокола), заголовков (headers) и тела запроса (body, необязательно).
  * HTTP-ответ состоит из: стартовой строки (версия протокола, статус-код, пояснение), заголовков и тела ответа (содержимое).
* **Более расширенный ответ**:
  * Заголовки (Headers) передают метаданные: тип кодировки, куки, токены авторизации (`Authorization`), тип отправляемого контента (`Content-Type`).
  * Тело запроса (Body) содержит отправляемые на сервер данные (например, JSON-строку при отправке формы через `POST` или `PUT`). В GET-запросах тело обычно отсутствует.
  * Тело ответа (Response Body) содержит данные, присланные сервером (HTML-код страницы, файл картинки или JSON-данные для фронтенда).

### 56. В чём разница между HTTP-методами GET и POST по способу передачи данных, что такое MIME Type и что означают группы статус-кодов 2xx, 4xx и 5xx? #http #network #api
* **Краткий ответ**: `GET` запрашивает данные, передавая параметры в виде query-параметров в строке URL. `POST` отправляет данные в теле (body) запроса. `MIME Type` указывает тип передаваемого контента.
  Группы кодов ответов:
  * `2xx`: успешная обработка запроса (200 OK).
  * `4xx`: ошибки со стороны клиента (404 Not Found, 401 Unauthorized).
  * `5xx`: ошибки со стороны сервера (500 Internal Server Error, 502 Bad Gateway).
* **Более расширенный ответ**:
  * GET-запросы кэшируются браузером, могут сохраняться в истории и имеют строгое ограничение по длине URL. POST-запросы не кэшируются, параметры скрыты в теле запроса, что делает их подходящими для отправки конфиденциальных данных и файлов любого размера.
  * `MIME Type` (Multipurpose Internet Mail Extensions) — строка, передаваемая в заголовке `Content-Type` (например, `application/json`, `text/html`, `image/png`), которая сообщает клиенту или серверу, как именно нужно парсить и обрабатывать тело сообщения.

### 57. Что такое метод fetch и формат JSON, и каково назначение методов JSON.stringify, JSON.parse и response.json()? #javascript #fetch #json #network
* **Краткий ответ**: `fetch` — это встроенный метод JS для выполнения асинхронных сетевых запросов, возвращающий промис. `JSON` — текстовый формат обмена данными.
  * `JSON.stringify(obj)` превращает JS-объект в JSON-строку.
  * `JSON.parse(str)` парсит JSON-строку в JS-объект.
  * `response.json()` — асинхронный метод fetch-ответа, возвращающий промис с распарсенным из JSON JS-объектом.
* **Более расширенный ответ**:
  * `fetch` возвращает промис, который разрешается в объект ответа `Response` сразу после получения заголовков от сервера.
  * Поскольку тело ответа может приходить частями, метод `response.json()` также асинхронен и возвращает промис.
  * Формат JSON (JavaScript Object Notation) является стандартом де-факто для обмена данными. Он требует двойных кавычек для ключей и строковых значений, и не поддерживает специфичные для JS типы вроде функций или значений `undefined`.

### 58. Что такое токены и эмбеддинги при работе языковой модели и за счёт чего модель формирует ответ? #ai #llm #general
* **Краткий ответ**: Токены — это минимальные текстовые единицы (части слов или символы), на которые модель разбивает текст. Эмбеддинги — числовые векторы, кодирующие семантический смысл токенов в многомерном пространстве. Модель формирует ответ путем вероятностного предсказания каждого следующего токена на основе входного контекста.
* **Более расширенный ответ**: Языковая модель (LLM) не может работать напрямую с текстовыми строками.
  1. Текст проходит токенизацию — разбиение на части слов (токены).
  2. Каждый токен преобразуется в эмбеддинг — массив чисел фиксированной длины, отражающий смысловое значение токена и его взаимосвязь с другими словами в языке.
  3. Нейросеть (обычно архитектуры Transformer) обрабатывает эти векторы через механизмы внимания (Attention) и вычисляет математическую вероятность того, какой именно токен должен логически следовать за текущим контекстом. Модель генерирует текст по одному токену за раз.

### 59. Что такое динамическая типизация в JavaScript и какие проблемы порождают слабая типизация и неявное приведение типов? #javascript #types #advanced
* **Краткий ответ**: Динамическая типизация означает, что тип переменной определяется в момент присвоения ей значения и может изменяться в ходе выполнения. Слабая типизация позволяет интерпретатору неявно приводить типы операндов при несовместимых операциях, что часто маскирует ошибки и вызывает скрытые баги.
* **Более расширенный ответ**: В JS можно написать `let a = 5; a = "text";` без ошибок. Слабая типизация приводит к неявному приведению типов (например, `5 + "5"` дает `"55"`, `[] + {}` дает `"[object Object]"`, а `null + 1` дает `1`). Это опасно тем, что при передаче неверного типа данных (например, от API или из формы) программа не упадет с ошибкой на раннем этапе, а продолжит работу с нелогичными значениями, что затрудняет поиск багов.

### 60. Что такое JSDoc и как он помогает сделать код самодокументированным ещё до перехода на TypeScript? #javascript #jsdoc #codestyle #typescript
* **Краткий ответ**: JSDoc — это стандарт оформления комментариев для документирования кода в JavaScript. С помощью специальных тегов (например, `@param`, `@returns`) он позволяет описывать типы аргументов и результатов функций, что дает IDE возможность выводить подсказки и проверять типы.
* **Более расширенный ответ**: JSDoc-комментарии оформляются в виде блоков `/** ... */` непосредственно перед функциями или классами.
  Пример:
  ```javascript
  /**
   * Вычисляет площадь прямоугольника.
   * @param {number} width - Ширина.
   * @param {number} height - Высота.
   * @returns {number} Площадь.
   */
  function getArea(width, height) { return width * height; }
  ```
  Современные среды разработки (VS Code) автоматически парсят JSDoc и показывают автодополнение и типы параметров при вызове функции. Также с помощью флага `// @ts-check` в начале файла можно заставить IDE подсвечивать нарушения типов на основе JSDoc.

### 61. Какие средства проверки типов во время исполнения есть в JavaScript? Объясните назначение typeof, Array.isArray() и instanceof. #javascript #types #runtime
* **Краткий ответ**:
  * `typeof`: проверяет базовые примитивные типы и функции (возвращает строку).
  * `Array.isArray(val)`: возвращает `true`, если переданное значение является массивом.
  * `instanceof`: проверяет, создан ли объект определенным конструктором (классом).
* **Более расширенный ответ**:
  * `typeof` имеет ограничения: например, `typeof null`, `typeof []` и `typeof {}` вернут `"object"`.
  * `Array.isArray(val)` специально создан для точного определения массивов, так как `typeof` не справляется с этой задачей.
  * Оператор `obj instanceof Constructor` проверяет, присутствует ли свойство `Constructor.prototype` в цепочке прототипов объекта `obj` (например, `myDate instanceof Date` вернет `true`).

### 62. Что нужно для добавления TypeScript в проект, какова роль файла tsconfig.json и что такое компиляция TS в JS? #typescript #tooling
* **Краткий ответ**: Нужно установить пакет `typescript` через NPM и создать файл конфигурации `tsconfig.json`. Компиляция TS в JS — это процесс трансляции исходного кода TypeScript в стандартный JavaScript с попутным удалением всех типов и проверок.
* **Более расширенный ответ**: Установка выполняется командой `npm install -D typescript`. Файл `tsconfig.json` указывает компилятору `tsc` правила сборки: целевую версию JS (`target`, например `ES6`), тип модульной системы (`module`), строгость проверок типов (`strict`), а также папки, которые нужно компилировать (`include`) или игнорировать (`exclude`). Поскольку браузеры не понимают синтаксис TypeScript, компилятор удаляет аннотации типов, интерфейсы и дженерики, оставляя чистый, готовый к исполнению JS-код.

### 63. Что такое строгий режим (strict) в TypeScript, зачем он нужен и что задают опции include и exclude? #typescript #advanced
* **Краткий ответ**: Строгий режим (`"strict": true` в `tsconfig.json`) включает максимальный уровень статической проверки типов.
  * `include`: массив glob-шаблонов путей к файлам, которые компилятор должен обрабатывать.
  * `exclude`: пути к файлам и папкам, которые компилятор должен игнорировать (например, `node_modules`).
* **Более расширенный ответ**: Включение строгого режима объединяет в себе несколько важных проверок, таких как `noImplicitAny` (запрет неявного вывода типа `any` для параметров функций) и `strictNullChecks` (запрет неявного использования `null` и `undefined` там, где ожидается конкретный тип). Это гарантирует максимальную надежность кодовой базы и предотвращает логические runtime-ошибки до этапа сборки.

### 64. Что такое приведение типов через as, чем оно отличается от настоящей проверки типа и какие риски несёт? #typescript #types #advanced
* **Краткий ответ**: Приведение типов через `as` (Type Assertion) — это ручное указание компилятору TypeScript рассматривать переменную как конкретный тип. Оно никак не влияет на runtime-код. Риск в том, что если тип значения в реальности окажется другим, в браузере произойдет падение с ошибкой, которую компилятор пропустил.
* **Более расширенный ответ**: Пример: `const input = document.querySelector('.input') as HTMLInputElement;`. Мы говорим компилятору: "Я гарантирую, что здесь будет инпут". Настоящая проверка типов в рантайме (например, через `if (input instanceof HTMLInputElement)`) физически проверяет объект в коде. А `as` — это лишь подсказка для статического анализатора TS. Если элемент окажется `null`, обращение `input.value` вызовет краш программы в рантайме, хотя TS не выдаст ошибок при сборке.

### 65. Что такое защитники типов (type guards) и чем кастомный защитник типа отличается от использования встроенных typeof/instanceof? #typescript #types #advanced
* **Краткий ответ**: Защитники типов — это выражения, позволяющие сузить тип переменной в блоке условий. Кастомный защитник — это пользовательская функция с возвращаемым типом-предикатом (`value is Type`), в отличие от встроенных операторов (`typeof`, `instanceof`), проверяющих только базовые типы.
* **Более расширенный ответ**: Встроенный защитник: `if (typeof value === "string") { value.toUpperCase(); }` — компилятор понимает, что внутри блока `value` гарантированно является строкой. Кастомный защитник пишется, когда нужно проверить сложную структуру объекта:
  ```typescript
  function isUser(obj: any): obj is User {
    return obj && typeof obj.id === 'number' && typeof obj.name === 'string';
  }
  ```
  Использование возвращаемого типа `obj is User` заставляет TS сузить тип проверяемой переменной до `User` везде, где функция вернула `true`.

### 66. Что такое дженерики (обобщённые типы), зачем они нужны и какую проблему решают по сравнению с использованием any? #typescript #generics #advanced
* **Краткий ответ**: Дженерики (Generics) — это параметры типов, позволяющие создавать функции, классы или интерфейсы, работающие с различными типами данных, сохраняя при этом строгую типизацию. Они решают проблему потери информации о типах, возникающую при использовании `any`.
* **Более расширенный ответ**: Если мы напишем функцию `function identity(arg: any): any { return arg; }`, компилятор забудет исходный тип переданного значения. С дженериком `function identity<T>(arg: T): T { return arg; }` тип возвращаемого значения динамически связывается с типом переданного аргумента. Если передать число, функция вернет строго тип `number`, что сохраняет автодополнение в IDE и безопасность типов.

### 67. Для чего используется extends в дженериках и как задать значение обобщённого типа по умолчанию? #typescript #generics #advanced
* **Краткий ответ**: Ключевое слово `extends` используется для ограничения (сужения) принимаемых дженериком типов. Значение по умолчанию задается с помощью знака `=` после объявления параметра типа.
* **Более расширенный ответ**:
  * Ограничение типов: `function logLength<T extends { length: number }>(arg: T) { console.log(arg.length); }`. Это гарантирует, что переданный аргумент обязательно имеет свойство `length`, иначе TS выдаст ошибку компиляции.
  * Значение по умолчанию: `interface Container<T = string> { value: T; }`. Если при создании объекта интерфейса тип `T` не указан явно, он автоматически примет тип `string`.

### 68. Что такое файлы декларации .d.ts и чем встроенная типизация библиотеки отличается от устанавливаемой (@types/...)? #typescript #tooling
* **Краткий ответ**: Файлы `.d.ts` (declaration) описывают сигнатуры типов JavaScript-кода без его реализации. Встроенная типизация поставляется внутри самой библиотеки, а устанавливаемая ставится отдельно из репозитория DefinitelyTyped (пакеты `@types/название`), если библиотека написана на чистом JS.
* **Более расширенный ответ**: Компилятору TS нужны типы для внешних библиотек. Если библиотека написана на TS, разработчики генерируют типы (`.d.ts`) при компиляции, и они поставляются прямо в npm-пакете (встроенная типизация). Если библиотека написана на JS, типов в ней нет. В этом случае сообщество пишет типы вручную и публикует их в репозитории DefinitelyTyped под именами `@types/react`, `@types/lodash` и т.д., и их нужно устанавливать отдельно как `devDependencies`.

### 69. Что такое React? В чем особенности использования ReactElement и React.Component #react #basics
* **Краткий ответ**: React — это библиотека для создания пользовательских интерфейсов на основе компонентов. `ReactElement` — это виртуальное представление DOM-элемента (неизменяемый объект), а `React.Component` — базовый класс для написания классовых компонентов.
* **Более расширенный ответ**: React использует декларативный подход и обновляет интерфейс при изменении данных.
  * **ReactElement** — это простая структура данных в памяти (JS-объект), описывающая DOM-узел (тип тега, пропсы, дети). Они создаются автоматически при вызове JSX.
  * **React.Component** — это классический базовый класс, предоставляющий методы жизненного цикла (`componentDidMount` и др.) и локальное состояние `this.state` для классовых компонентов. В современной разработке функциональные компоненты с хуками практически полностью вытеснили классовые компоненты, но поддержка классов сохраняется.

### 70. Что такое JSX? Приведите пример кода, иллюстрирующий его использование. #react #jsx #basics
* **Краткий ответ**: JSX (JavaScript XML) — это расширение синтаксиса JavaScript, которое позволяет писать HTML-подобную разметку непосредственно внутри JS/TS файлов.
* **Более расширенный ответ**: JSX упрощает создание структуры компонентов, делая код более наглядным. Под капотом сборщик (Babel/Vite) компилирует JSX в вызовы функций `React.createElement(...)`.
  Пример:
  ```jsx
  const title = "Заголовок страницы";
  const element = <h1 className="main-title">{title}</h1>;
  ```
  После компиляции это превращается в:
  ```javascript
  const element = React.createElement('h1', { className: 'main-title' }, title);
  ```

### 71. Поясните понятие виртуальной DOM. Чем она отличается от реальной и для чего используется? #react #virtual-dom #advanced
* **Краткий ответ**: Виртуальный DOM (Virtual DOM) — это легковесная копия реального DOM, хранящаяся в оперативной памяти в виде JS-объектов. Она используется для оптимизации производительности: React вносит изменения сначала в неё, вычисляет разницу (diffing) и обновляет в реальном DOM только изменившиеся участки.
* **Более расширенный ответ**: Прямое изменение реального DOM — ресурсозатратная операция, так как любое изменение дерева заставляет браузер заново пересчитывать стили и геометрию элементов (процессы Reflow и Repaint). В виртуальном DOM операции происходят мгновенно. React при изменении данных строит новое виртуальное дерево, сравнивает его со старым (алгоритм reconciliation) и формирует минимальный набор точечных изменений, который затем за один проход применяется к реальному DOM, значительно ускоряя работу интерфейса.

### 72. Что такое Babel? Поясните назначение и особенности использования Babel-cli. #tooling #babel #react
* **Краткий ответ**: Babel — это транспайлер, который преобразует современный JavaScript (ES6+) и JSX в обратно-совместимый JS-код для старых браузеров. `Babel-cli` — это консольный инструмент, позволяющий запускать транспайлинг файлов напрямую из командной строки.
* **Более расширенный ответ**: Babel преобразует синтаксис (например, стрелочные функции в обычные, классы в прототипы) и добавляет полифилы для отсутствующих методов. `babel-cli` (устанавливается как `@babel/cli`) позволяет выполнять компиляцию файлов через терминал с помощью команды `npx babel src --out-dir dist`. В больших проектах Babel обычно не запускают через CLI вручную, а встраивают в сборщики (Vite, Webpack) через лоадеры и плагины.

### 73. Поясните концепцию SyntheticEvent в React #react #events #advanced
* **Краткий ответ**: `SyntheticEvent` — это кроссбраузерная обертка React вокруг нативного объекта события браузера. Она стандартизирует поведение событий, гарантируя их одинаковую работу во всех браузерах.
* **Более расширенный ответ**: React перехватывает нативные события браузера и оборачивает их в объект `SyntheticEvent`. Этот объект имеет те же методы (`preventDefault()`, `stopPropagation()`), что и нативные события, но решает все проблемы кроссбраузерной несовместимости. Начиная с React 17, пулинг событий (обнуление свойств события для экономии памяти) отключен, что позволяет безопасно использовать объект события в асинхронном коде. Доступ к оригинальному событию браузера можно получить через свойство `event.nativeEvent`.

### 74. За что отвечают props и state для объекта React? #react #state #props
* **Краткий ответ**:
  * `props` (пропсы) — входные данные, передаваемые компоненту извне (от родительского компонента). Они неизменяемы (read-only).
  * `state` (состояние) — внутренние изменяемые данные компонента, которые он контролирует сам. Изменение любого из них вызывает автоматический ререндер.
* **Более расширенный ответ**:
  * Компонент принимает `props` как параметры и не имеет права изменять их: они передаются декларативно в JSX как атрибуты (например, `<Card title="Привет" />`).
  * `state` представляет собой локальное хранилище данных компонента для реактивного управления (например, введенный текст в инпут, состояние открытого модального окна). В функциональных компонентах состояние создается через хук `useState`, а в классовых — через свойство `this.state` и метод `this.setState`.

### 75. Что такое жизненный цикл компонентов-классов? #react #lifecycle #advanced
* **Краткий ответ**: Жизненный цикл — это последовательность этапов, которые проходит компонент от момента его создания до удаления из DOM-дерева. Он состоит из трех основных фаз: Монтирование (Mounting), Обновление (Updating) и Размонтирование (Unmounting).
* **Более расширенный ответ**: На каждом этапе вызываются специальные методы:
  * **Монтирование**: `constructor()` -> `render()` -> `componentDidMount()` (вызывается один раз, подходит для сетевых запросов и подписок).
  * **Обновление** (при изменении state или props): `render()` -> `componentDidUpdate(prevProps, prevState)` (реагирует на изменения данных).
  * **Размонтирование**: `componentWillUnmount()` (вызывается перед удалением компонента, подходит для очистки таймеров и удаления слушателей).

### 76. Что такое Context API в React? #react #context #state-management
* **Краткий ответ**: Context API — это встроенный механизм React для передачи данных глубоко по дереву компонентов без необходимости передавать пропсы вручную через каждый промежуточный уровень (решает проблему prop drilling).
* **Более расширенный ответ**: Context создается с помощью функции `React.createContext()`. Он предоставляет компонент `Provider`, который оборачивает родительское дерево и принимает проп `value`. Любой дочерний компонент на любом уровне вложенности может получить доступ к этому значению (в функциональных компонентах — с помощью хука `useContext`). Это удобно для хранения глобальных данных: текущей темы оформления, языковой локали или состояния авторизации пользователя.

### 77. Поясните назначение и особенности модуля React-Router #react #routing
* **Краткий ответ**: `React-Router` — это библиотека для организации маршрутизации (навигации) в React-приложениях. Она связывает URL-адрес в адресной строке с отображением конкретных компонентов без перезагрузки всей страницы (SPA-подход).
* **Более расширенный ответ**: Библиотека перехватывает клики по ссылкам и динамически обновляет DOM-дерево.
  Основные элементы:
  * `BrowserRouter`: компонент-обертка, связывающий приложение с историей браузера (HTML5 History API).
  * `Routes` и `Route`: сопоставляют путь (`path`) с соответствующим компонентом (`element`).
  * `Link` и `NavLink`: компоненты для создания навигационных ссылок без перезагрузки вкладки.
  * Библиотека также предоставляет хуки `useNavigate` для программного изменения URL и `useParams` для извлечения динамических параметров маршрута (например, `id` из пути `/user/:id`).
