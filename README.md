# Vibe Prep — Платформа Подготовки к Экзаменам

Умное веб-приложение для эффективной подготовки к экзаменам, зачетам и собеседованиям на основе флэш-карт и интервального повторения. 

**Живая демо-версия**: [https://rimuwu.github.io/vibe-preparation/](https://rimuwu.github.io/vibe-preparation/)

---

## 🚀 Основные возможности

1. **📚 Обучение по разным темам (Модули)**
   * Разделение вопросов на независимые темы (модули).
   * Стандартный модуль по дисциплине **«Фронтенд-разработка»** по умолчанию находится в папке `modules/`.
   * Возможность загружать свои собственные темы (файлы `.md`), указывать для них названия/описания и сохранять их в памяти приложения без риска потерять при перезагрузке.
   * Возможность удалять добавленные кастомные темы в один клик.

2. **💾 Сохранение сессии (Персистентность)**
   * Состояние прохождения текущего теста автоматически сохраняется в `localStorage` при каждом действии.
   * При случайной перезагрузке страницы или закрытии вкладки вы вернетесь ровно к тому же вопросу, с сохранением всех очков и прогресса текущей сессии.

3. **↩️ Возврат к прошлому вопросу**
   * Передумали или ошиблись кнопкой? Кнопка **«Назад»** (или клавиша `Backspace`) вернет вас к предыдущему вопросу.
   * Статистика текущей сессии (число попыток и правильных ответов) автоматически корректируется при переответе.

4. **★ Режим «Только сложные вопросы»**
   * Возможность тренировать только те карточки, которые вызывают затруднения.
   * Включает в себя как вопросы, помеченные вручную кнопкой *«Сложный вопрос»*, так и вопросы со специальной меткой `[!]` в исходном `.md` файле.

5. **📊 Синхронизация прогресса**
   * **Локальный бэкап**: Экспортируйте всю историю ответов и пользовательские темы в файл `vibe-prep-progress-YYYY-MM-DD.json` и импортируйте его на любом другом девайсе.
   * **Облачная синхронизация**: Свяжите устройства с помощью уникального кода `VIBE-XXXX-YYYY`. Прогресс будет автоматически выгружаться в облако (MongoDB Atlas) и загружаться на связанных девайсах.

6. **🧠 Три режима проверки знаний**
   * **Свободный ответ** (устное самооценивание).
   * **Выбор варианта** (интерактивный тест с дистракторами из базы).
   * **Ввод текста** с умным алгоритмом проверки (fuzzy matching, расстояние Левенштейна и поддержка запоминания ваших верных альтернативных ответов).

7. **🏆 Онлайн-таблица лидеров**
   * Соревнуйтесь с другими участниками по стандартным учебным модулям.
   * Рейтинг строится по количеству пройденных вопросов, точности ответов и времени последнего обновления.

8. **🛡️ Защита от накрутки (Anti-Abuse)**
   * Таблица лидеров привязана к коду синхронизации. Один код — один профиль на таблице лидеров.
   * Для предотвращения спама аккаунтами ограничена возможность генерации кодов синхронизации — не более 2 кодов на одно устройство (проверка по уникальному SHA-256 цифровому отпечатку браузера IP + User-Agent).

## 🛠️ Запуск проекта локально

Проект состоит из клиентской части (чистый HTML/JS/CSS) и серверной части (FastAPI для синхронизации и лидерборда с БД MongoDB).

### Запуск фронтенда
1. Установите зависимости:
   ```bash
   npm install
   ```
2. Запустите локальный сервер:
   ```bash
   npm run dev
   ```
3. Откройте `http://localhost:3000` в браузере.

### Запуск бэкенда
1. Перейдите в директорию `backend`:
   ```bash
   cd backend
   ```
2. Создайте файл `.env` в корне проекта на основе `.env.example`, указав вашу строку подключения к MongoDB Atlas:
   ```env
   MONGODB_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/vibe_prep
   ```
3. Установите зависимости Python:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите API-сервер:
   ```bash
   python -m uvicorn main:app --host 127.0.0.1 --port 8000
   ```
   Фронтенд автоматически переключится на локальный бэкенд при запуске с `localhost`.

## 📂 Структура проекта

```text
vibe-preparation/
├── backend/              # Серверная часть (API)
│   ├── main.py           # Логика FastAPI, хэширование fingerprint, эндпоинты
│   ├── requirements.txt  # Зависимости Python (FastAPI, PyMongo и др.)
│   ├── vercel.json       # Конфигурация для деплоя бэкенда на Vercel
│   └── DEPLOY.md         # Руководство по развертыванию
├── modules/
│   ├── fr.md             # База вопросов по фронтенд-разработке (стандартная)
│   └── modules.json      # Описания стандартных тем
├── FORMATTING.md         # Руководство по разметке markdown-файлов
├── README.md             # Настоящее руководство
├── index.html            # Главная разметка приложения (все экраны)
├── styles.css            # Дизайн-система (Google Material Dark)
├── app.js                # Основная логика приложения и управления экранами
├── parser.js             # Модуль парсинга markdown-файлов в JSON-объекты
├── stats.js              # Модуль работы с localStorage и бэкапами
├── algorithms.js         # Алгоритмы фильтрации и сортировки вопросов
├── config.json           # Публичный конфиг со ссылкой на прод-сервер
└── package.json          # Конфиг npm скриптов
```
